# WhyBook — Chemistry Context Companion for Offline Learners
> "Education should be free and accessible to everyone."

---

## The Problem This Solves

250 million students in India study on the NCERT curriculum. Most of them are in Tier 2-3 cities and towns where:
- Internet is 2G or unreliable
- Private tutors are unaffordable
- Textbooks exist — but no one explains *why* anything in them matters

Students memorize CH₄, H₂SO₄, and the periodic table to pass exams. They never learn that methane is the gas in their kitchen cylinder, or that sulfuric acid is in the battery of their father's motorcycle. The gap isn't access to information — it's the absence of context.

This is not a hypothetical user. This was the builder of this project, in 2018.

---

## The Solution

A Kaggle notebook application where a student photographs any NCERT Chemistry page →
Gemma 4 E4B (fine-tuned on NCERT curriculum) identifies the primary concept and returns:

1. **What it means** — in simple terms, no jargon
2. **Why it's in the textbook** — the pedagogical purpose
3. **Where they'll encounter it in real life** — grounded, specific, local

Output is in the student's language — Hindi, Marathi, Bengali, Tamil, or any of 140 languages — with zero internet required after setup.

## Gemma 4 Features Used

| Feature | How |
|---|---|
| **Vision input** | Student photographs NCERT textbook page; E4B identifies the primary formula/concept from the image |
| **On-device E4B** | Model quantized to GGUF; runs on Android device with no internet after one-time download |
| **140-language output** | Base E4B handles Hindi, Marathi, Bengali, Tamil natively — no retraining needed for language |
| **Fine-tuned on domain data** | NCERT Chemistry Class 9-12 (public domain PDFs) → synthetic Q&A via base Gemma 4 → QLoRA 4-bit fine-tune via Unsloth + PEFT |
| **Quantization** | GGUF post fine-tune — runs on Kaggle T4 for demo, deployable on Android |

**Not in scope:** Audio input, function calling, long context (not needed for single-page photo use case — keep it clean)

## Technical Stack

```
Model:         Gemma 4 E4B
Fine-tuning:   QLoRA 4-bit via Unsloth + PEFT
Training data: NCERT Chemistry PDFs (Class 9-12, public domain)
               → synthetic Q&A generated by base Gemma 4
               → format: { concept, what, why, real_world_example }
Compute:       Kaggle 2×T4 (free, 30hrs/week) — training + demo + submission
Quantization:  GGUF (llama.cpp) for on-device deployment
Inference:     Hugging Face Transformers in Kaggle notebook
Frontend:      Gradio widget in Kaggle notebook (photo upload + language select + output)
License:       Apache 2.0 (Gemma 4) — fully open, no restrictions
```

**Everything runs on Kaggle free tier. Zero paid compute. Zero paid APIs.**

## Fine-Tuning Data Strategy

1. Download NCERT Chemistry PDFs (Class 9, 10, 11, 12) — fully public domain
2. Extract text + identify concepts (formulas, reactions, elements, compounds)
3. Use base Gemma 4 E4B to generate synthetic Q&A pairs per concept:
   ```
   Input: [concept/formula from page]
   Output:
     what: "..."
     why: "..."
     real_world: "..."
   ```
4. Review + filter for quality (remove hallucinations, vague answers)
5. Fine-tune E4B on cleaned pairs using QLoRA 4-bit + Unsloth on Kaggle 2×T4
6. Quantize to GGUF

## Why Now — Why Gemma 4 Specifically

Before Gemma 4, this required:
- Cloud inference (needs internet → kills the offline use case), OR
- A model too large to fine-tune on free hardware (kills the zero-cost constraint)

Gemma 4 E4B is the first open model that combines:
- **Vision input** (photograph the textbook)
- **140-language output** (no retraining per language)
- **On-device feasibility** (GGUF on Android)
- **Fine-tunable on free hardware** (QLoRA on Kaggle T4)

All four at once. That combination did not exist before April 2026.
